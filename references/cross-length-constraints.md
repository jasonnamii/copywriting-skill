## Compression Techniques

### Korean-Specific (자 compression)

#### 1. 조사 생략 (Omit particles)
- **Normal:** "우리가 만든 도구는 정말 빠르다" (17자)
- **Compressed:** "우리 도구 정말 빠르다" (11자) [30% shorter]
- **Rule:** When meaning is clear, drop 이/가, 을/를, 에, 의

#### 2. 주어 생략 (Omit subject)
- **Normal:** "당신이 이 도구를 쓰면 시간을 절약할 수 있다" (20자)
- **Compressed:** "이 도구로 시간 3배 절약" (11자) [45% shorter]
- **Rule:** If subject is obvious (you, us), remove it

#### 3. 한자어 활용 (Use hanja for density)
- **Normal:** "빨리 빨리" (4자)
- **Compressed:** "신속" (2자) [50% shorter]
- **Rule:** Replace 순우리말 (native Korean) with 한자어 (hanja) where formal authority helps

#### 4. 동사 축약 (Abbreviate verbs)
- **Normal:** "설정을 할 필요가 없다" (10자)
- **Compressed:** "설정 불필요" (5자) [50% shorter]
- **Rule:** 하다 → X or replace with noun form

#### 5. 수치 단순화 (Simplify numbers)
- **Normal:** "오천사백오십 명" (9자)
- **Compressed:** "5.4k명" (4자) [55% shorter]
- **Rule:** Use abbreviated numbers (k, m, %) when acceptable

### English-Specific (character compression)

#### 1. Active Voice (reduce words by 15-20%)
- **Passive:** "Features were removed by us to speed up setup" (44 chars)
- **Active:** "We removed bloat. Setup now 3x faster." (38 chars) [14% shorter]

#### 2. Verb-First Structure
- **Normal:** "You can save 3 hours per week" (31 chars)
- **Compressed:** "Save 3 hrs/week" (16 chars) [48% shorter]
- **Rule:** Start with action, omit object if implied

#### 3. Abbreviations (use sparingly)
- **Normal:** "without credit card required" (27 chars)
- **Compressed:** "no CC req'd" (11 chars) [59% shorter]
- **Rule:** Use abbreviations if audience familiar (CC, min, hrs, sec, etc.)

#### 4. Compound Removal (split sentences)
- **Normal:** "We built a fast, reliable, easy-to-use tool" (44 chars)
- **Compressed:** "Fast. Reliable. Simple." (23 chars) [48% shorter]
- **Rule:** Replace compound adjectives with fragment lists

#### 5. Number Simplification
- **Normal:** "more than twenty-five percent" (30 chars)
- **Compressed:** "25% faster" (10 chars) [67% shorter]
- **Rule:** Use symbols %, +, =, > wherever possible

#### 6. Contraction & Casual Forms
- **Normal:** "You will see the results immediately" (35 chars)
- **Compressed:** "You'll see results instantly" (28 chars) [20% shorter]
- **Rule:** Use you'll, we've, it's, they're over formal equivalents

## Length × Tone Interaction

### Tone Affects Ideal Length

| Tone | Optimal Length Modifier | Reason |
|------|--------|---------|
| **High Casual** | -15% (shorter) | Brevity reads more natural; longer feels stilted |
| **High Funny** | -20% (much shorter) | Jokes lose power if explained too long |
| **High Irreverent** | -10% (slightly shorter) | Rule-breakers move fast; lengthy = establishment |
| **High Enthusiastic** | +10% (slightly longer) | Superlatives need explanation; energy sustains longer copy |
| **High Serious** | +15% (longer) | Authority requires detail; brevity reads dismissive |
| **High Formal** | +10% (slightly longer) | Complete sentences natural; fragments look careless |

### Example: C6 Social Post (Typical: 80-250 chars)

**C8 Mode (High Casual, High Funny, High Enthusiastic):**
- Optimal: 150 chars (shorter, punchier, joke works tighter)
- "we removed 12 features nobody asked for. setup now takes 3 min, not 45. that's what happens when you care."

**Universal Mode (Moderate Casual, Low Funny, Moderate Enthusiastic):**
- Optimal: 220 chars (slightly longer, more explanation)
- "We redesigned our onboarding based on 6 months of user feedback. Setup is now 3 minutes instead of 45. Try it:"

## Length × Copy Type Benchmark: When You're Unsure

**If in doubt about length, use this decision tree:**

```
START: What copy type?

A-types (App)? 
  → A5/A7/A10 (quick actions) = 20-40 chars
  → A1/A2/A3/A4/A6/A8 (engagement) = 60-100 chars
  → A9 (help) = 150-250 chars
  
B-types (Strategy)?
  → B2/B5 (quick articulation) = 80-100 chars
  → B1/B4 (reasoning) = 150-300 chars
  → B3 (narrative) = 300-600 chars
  
C-types (Marketing)?
  → C1/C4/C5/C7/C9/C10 (hook) = 30-100 chars
  → C3/C6 (hero + proof) = 100-300 chars
  → C2 (full story) = 300-600 chars
  → C8 (creator deep-dive) = 150-400 chars
  
D-types (CRM)?
  → D2 (re-engagement nudge) = 150-300 chars
  → D1/D3 (full sequence) = 400-800 chars
```

## Compression Intensity Scale

Rate your compression need 1-5:

| Level | When | Technique | Example |
|-------|------|-----------|---------|
| **1 (None)** | Plenty of space (B3, D1) | Write naturally | "We built this over 6 months..." |
| **2 (Light)** | Normal constraints (C2, C3) | Remove 1 adjective, use contractions | "We built this tool in 6 months..." |
| **3 (Medium)** | Tight constraints (C4, C6) | Aggressive contractions, abbrev., fragments | "Built in 6mo. It ships faster." |
| **4 (Heavy)** | Severe limits (C1, push) | All tactics + number simplification | "Built in 6mo. 3x faster." |
| **5 (Extreme)** | Platform maximum (push 32 chars) | Korean: 주어생략, 조사생략. English: symbols | "6mo build. 3x↑" |
