# Experimental Tone Guide

Breaking the mold. For types where hip can go further. Benchmarks: Duolingo, 배달의민족, Arc build-in-public, Wendy's.

**ALLOWED TYPES ONLY:** A1, A2, A6, A9, A10, C1, C5, C7, D3
Not for: A3 (setup), A4 (pricing clarity), A5 (testimonials), A7 (onboarding), A8 (advanced), A8 (community core), B types (strategy), D1 (support), D2 (legal)

---

## Experimental Tone DNA

### 4th Wall Breaks
- Address the audience as co-conspirators. "We know you know this is an ad."
- Admit limitations with humor: "Yeah, we're not perfect. Yet."
- Flip expectations: "Every other tool says this. We're not lying though."

### Format Subversion
- Use unexpected structure. Bullet lists that break mid-thought. Paragraphs that—
- Fragment sentences. For effect. Trust the reader.
- Layout rules are suggestions. Use whitespace like punctuation.
- Emoji as verb, not decoration: "This design ➡️ code in 30 min" (emoji shows transformation)

### Meme Grammar
- Invert syntax for rhythm: "Code, we write. You don't."
- Stack concepts: "Speed. Quality. Price. Pick two. (We picked all three.)"
- Repetition as emphasis: "Fast fast fast."
- Casual contraction: "ain't," "y'all," "gonna" — but only if natural to the regional voice

### Cultural Reference (Creator-Specific)
- Duolingo: Absurdist mascot, fourth-wall breaks, self-aware
- 배달의민족: Playful character voices, meme formats, inside jokes
- Arc: Build-in-public transparency, "we're figuring it out," failure as content
- Wendy's: Roast competitors without being mean, own your weirdness

### The Risk Zone
Experimental can feel **immature, dismissive, or try-hard** fast. Rules:

1. **Don't preach.** Experimental voice should feel like discovery, not lecture.
2. **Don't poke fun at the user.** Only poke fun at the industry, competitors, or yourselves.
3. **Emoji frequency:** 1 per 50 words max. Every sentence with emoji feels desperate.
4. **Slang has shelf life.** "Slay," "no cap," "it's giving" age fast. Use sparingly.
5. **Avoid forced memes.** "Why did the designer cross the road?" — no. Just no.

## Copy Examples by Type

### A1: Feature Overview (EXPERIMENTAL)

**Korean Examples:**
1. "코드로 바뀌는 디자인. 마법이 아니라 기술. (둘 다 믿기 어렵지.)"
2. "Figma → HTML/CSS. 중간 과정 생략. 당신이 하던 수작업은 이제 역사."
3. "당신의 디자인이 말한다: 'Code me, bro.'"

**English Examples:**
1. "Design → Code. Not magic. Technology. (We don't blame you for being skeptical.)"
2. "Everything between Figma and GitHub? We deleted it."
3. "Your design, to a developer: 'Here's the code. You're welcome.'"

**Anti-pattern (went too far):**
❌ "C8 yeets your design into the JavaScript void and out pops perfection lmao no cap fr fr"

**Risk Assessment:**
- **Low risk:** Skepticism + confidence = relatable
- **Medium risk:** Deleting the middle = seems arrogant if code quality isn't there
- **High risk:** Fourth-wall break only works if the product backs it up

### A2: Value Proposition (EXPERIMENTAL)

**Korean Examples:**
1. "4시간이 30분이 되는 마법. (진짜로. 마법이 아니라 API, 근데 마법처럼 보여.)"
2. "당신의 야근을 없애기 위해 우리가 밤을 샌 거예요."
3. "시간 낭비? 우리가 없앴어. 돈 낭비? 우리가 있어. But worth it."

**English Examples:**
1. "Four hours becomes 30 minutes. (Not actually magic. Just code that feels like it.)"
2. "We lost sleep so you wouldn't have to."
3. "Wasted time: removed. Wasted money: gone. Time left for actual design: infinite."

**Anti-pattern (went too far):**
❌ "We basically rewrote the laws of physics using dark magic and vibes, bestie"

**Risk Assessment:**
- **Low risk:** Sleep jokes = everyone gets it
- **Medium risk:** Claims of removing time entirely = overpromising
- **High risk:** Sarcasm about your own product reads as not believing in it

### A6: Error States / Edge Cases (EXPERIMENTAL)

**Korean Examples:**
1. "너무 커. 파일 크기를 줄이면 우리도 일할 수 있어."
2. "이 파일? C8이 읽지 못하는 언어로 쓰여 있어. (농담 아니야. 진짜 이상해.)"
3. "뭔가 깨졌나요? 우리 팀이 이미 캐치했으면 좋겠는데. 이메일 보내주면 진짜로 밤을 새울게."

**English Examples:**
1. "File's too big. We can fix that. Just... not like this."
2. "This format. C8 doesn't speak it. (Weird. We checked twice.)"
3. "Something broke. We probably already know. Email anyway. We're into pain."

**Anti-pattern (went too far):**
❌ "ur file made C8 have an existential crisis bestie we're in shambles rn"

**Risk Assessment:**
- **Low risk:** Admitting limitation = honesty, builds trust
- **High risk:** Making errors feel funny might bury the actual solution
- **High risk:** Self-deprecation on errors can feel like "we don't care"

### A10: Call-to-Action (EXPERIMENTAL)

**Korean Examples:**
1. "지금 시작해. 신용카드 필요 없어. 30초 후에 '어? 진짜 되네?'라고 하게 될 거야."
2. "한 번만 써봐. 우리가 아니라 코드가 말해줄 거야."
3. "당신의 선택: 밤 2시에 시작 vs 지금 시작. C8 없이 vs C8이랑."

**English Examples:**
1. "Try it. No card. You'll be like 'wait it actually works?'"
2. "Let the code speak for itself. We'll shut up."
3. "Your choice: Start at 2 AM or 6 PM. Today or yesterday."

**Anti-pattern (went too far):**
❌ "YEET yourself into the future by CLICKING this button BRUH no cap the vibes are immaculate"

**Risk Assessment:**
- **Low risk:** Confidence + self-deprecation
- **Medium risk:** Directness without being pushy
- **High risk:** Meta-jokes only work if the product is clearly worth it

## A/B Testing Slots

Which types are safest to experiment with:

| Type | Experimental Level | Notes |
|------|-------------------|-------|
| A1 (Overview) | Medium | Feature confidence matters |
| A2 (Value Prop) | Medium-High | Claims must be backed up |
| A6 (Errors) | Low | Clarity + humor = good |
| A9 (Community) | Medium | Inclusion > hype |
| A10 (CTA) | High | Confidence naturally reads experimental |
| C1 (Campaign) | Medium | Depends on audience expectations |
| C5 (Content) | Low-Medium | Educational requires some trust |
| C7 (Success) | Low | Understate > overstate |
| D3 (Billing) | Low | Safety first, humor second |

**Safe experiments:**
- A6 + error states (admitting limitations)
- A10 + CTAs (conversational tone)
- C1 + campaigns (breaking expectations is the point)

**Risky experiments:**
- A2 + promises (credibility >> cleverness)
- A4 + pricing (clarity required)
- C7 + social proof (facts > fun)
- D1 + support (empathy > jokes)

## Format Subversion Examples

### Fragment Sentences
Good: "Speed matters. So does quality. We nailed both."
Bad: "Speed and quality are synergistically optimized."

### Whitespace as Punctuation
```
Here's what C8 does:

Takes your design.

Writes the code.

You ship.

That's it.
```

### Bullet Lists That Break Format
- Design to code
- No bullshit
- Actually works
- ...for real this time
- (past products didn't. we know.)

### Inversion for Rhythm
"What makes code fast? Speed. What makes speed safe? Testing. We do both, so you don't choose."

## Emoji as Verb (Not Decoration)

Use emoji to **show transformation or action**, not just color:
- "Design → ➡️ Code" (clear transformation)
- "You ➡️ Shipped" (you becoming the verb)
- "Figma ➡️ GitHub" (workflow, not aesthetic)

Avoid:
- Emoji in every sentence ("We're so ✨ vibes ✨")
- Emoji as reaction ("That's so cool 🔥")
- Emoji as decor ("Check out this amazing feature ☺️")

---

## When Experimental Goes Too Far

Red flags:
1. **Forced slang.** "C8 is lowkey bussin" = sounds like a dad at a school dance
2. **Mocking the user.** "lol if you can't use C8 you're dumb" = not funny, toxic
3. **Over-explanation.** If you have to explain the joke, it's not working
4. **Inconsistent voice.** Experimental in one sentence, formal the next = jarring
5. **Sacrificing clarity.** Clever ≠ confusing. Ever.

---

## Usage Notes

- Every example is product-ready (tested in landing page, social, email)
- Korean uses contemporary, natural slang (not forced)
- English uses Gen-Z sensibility (but understatement > excess)
- Experimental works best in **social, ads, and communities** — not in onboarding
- A/B test experimental against standard before launch
- If team is uncomfortable with tone level, dial back one notch
