# UX_WRITING_MODE — A·D계열 필수 마이크로카피 체크

**허브:** `ux-principles.md#A-N2,N5,N9,N10`. 이 모드는 N2·N5·N9·N10 코어 4원리 적용.

**발동:** A계열(A1~A10 앱 카피) · D계열(D1~D3 CRM) · C계열 내 에러·안내 카피.
**스킵:** B계열(문서)·C계열 내 브랜딩/마케팅.

---

## 4원리 마이크로카피 체크

| # | 원리 | 카피 체크 | 위반 패턴 | 수정 방향 |
|---|------|----------|---------|---------|
| N2 | MATCH_REAL | 사용자 언어인가 (시스템 용어·전문용어 ✗) | "오류코드 500" | "지금은 연결이 불안정해요" |
| N5 | ERR_PREV | 실수 전 경고·확인하는가 | "삭제" 단독 버튼 | "이 계정과 연결된 모든 데이터가 사라집니다. 삭제하시겠어요?" |
| N9 | ERR_REC | 오류시 원인·영향·해법 3요소 있는가 | "오류가 발생했습니다" | "비밀번호가 일치하지 않아요. 다시 입력하거나 [비밀번호 찾기]를 눌러주세요" |
| N10 | HELP | 도움말 즉시 접근 가능한가 | 막다른 에러 | "문제가 계속되면 [1:1 문의]로 알려주세요" |

---

## 에러 메시지 3요소 (N9 필수)

```
① 원인(무엇이): 비밀번호가 일치하지 않아요
② 영향(무슨 일): — (이 경우 생략 가능)
③ 해법(어떻게): 다시 입력하거나 [비밀번호 찾기]를 눌러주세요
```

**3요소 중 1+ 누락 = FAIL.** 원인만 적고 해법 누락 = 가장 흔한 위반.

---

## 톤과의 관계

- 🟡 Experimental 차단 맥락(에러·법적·결제)은 §1 축2 규칙대로 ⚪ Standard 강제 — UX_WRITING_MODE는 그 **위에** 4원리 검증 추가.
- 🔵 Hip 톤에서도 N9 3요소는 유지 — Hip한 에러 메시지도 원인·해법은 명확해야.

## 🚪 게이트

A·D계열 카피 산출시 4원리 모두 PASS → §5 QC 진행. 1개+ FAIL → 해당 카피 재작성.
