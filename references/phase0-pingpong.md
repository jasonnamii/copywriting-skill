# Phase 0 — 5축 상황 핑퐁 (상세)

**목적:** 카피 생성 전 맥락 5축을 수집해 엉뚱·일반함 원인을 차단.

## 5축 정의

| # | 축 | 정의 | 예시 | 등급 |
|---|---|---|---|---|
| 1 | 누구에게 | 타겟의 구체 장면 1줄 | "월요일 9시 출근길 30대 직장인" | 필수 |
| 2 | 어디서 | 채널·매체 | 푸시·랜딩헤드라인·이메일제목·인스타·옥외 | 필수 |
| 3 | CTA | 다음 행동 1개 | 클릭·구독·공유·기억·결제 | 필수 |
| 4 | 직전 상태 | 카피 직전 사용자 행동 | 피드 스크롤 vs 결제 고민 | 선택 |
| 5 | 브랜드 진실 | 경쟁사가 못 하는 우리만의 관찰 | (예: "재택SaaS만 가능한 9시 늦출근") | Hip 필수·범용 선택 |

## 핑퐁 질문 템플릿 (결손축에만)

- **누구:** "이 카피를 받을 사람의 구체 장면을 1줄로? 예: '월요일 9시 출근길 30대 직장인'"
- **어디:** "어떤 매체? 푸시·랜딩헤드라인·이메일제목·인스타·옥외 중?"
- **CTA:** "받은 사람이 뭘 하길 원해요? 클릭·구독·공유·기억·결제 중 1개?"
- **직전:** "직전에 사용자는 뭘 하고 있었을까요? (선택)"
- **진실:** "경쟁사는 못 하고 우리만 할 수 있는 관찰 1개? (Hip 필수)"

## fast-path 예외

| 조건 | 처리 |
|---|---|
| 입력에 필수 3축 텍스트 명시 | 핑퐁 스킵·즉시 §1 진입 |
| "예시·샘플·테스트로" 명시 | 핑퐁 스킵, 출력 상단 "📌 추정값" 명시 |
| 진단 모드 | "이 카피의 의도된 5축이 뭐였나요?" 1회 |
| Hip톤 요청 | fast-path 적용 ✗ — 진실축 항상 핑퐁 |

## 핑퐁 실행 흐름

```
입력 받음
  ├─ 필수 3축 충족? → fast-path 판정
  └─ 미충족 → AskUserQuestion으로 결손축만 묶어 1회 핑퐁
       ├─ 답변 수집 → 5축 확정 → §1 진입
       └─ 거부·스킵 → 추정값 명시("추정: ~로 가정함, 다를 시 알려주세요") 후 진행
```
