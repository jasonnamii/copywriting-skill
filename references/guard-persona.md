# Guard: Persona Segmentation (Creator Focus + Universal Modes)

**Core Principle:** "Creator" is not monolithic. A designer reads copy differently than a developer. Video creators respond to energy that alienates writers. Persona detection prevents misfires—choosing the right tone for the right person.

---

## C8 Mode: 4 Primary Creator Personas

### Persona 1: Designer (Visual, Typographic, Figma-Native)

**Demographics:**
- Age: 24-45, 70% female, global
- Tools: Figma, Framer, Adobe CC, Webflow
- Income: $50K-$150K+ (freelance to staff)
- Location: Urban hubs (NYC, London, Berlin, Seoul, Tokyo, SF)

**Psychographics:**
- Thinks in systems (design tokens, spacing scales, color theory)
- Perfectionist about aesthetics; pragmatic about speed
- Follows design trends (Dribbble, Design Observer)
- Community-oriented (design Twitter, conferences)
- Craves validation (likes/retweets on design shares)

**Copy Preferences:**
- Length: Short, scannable (5-10 words ideal)
- Vocabulary: Design-specific (layout, hierarchy, systems, components)
- Tone: Aspirational + practical (beauty meets function)
- Humor: Visual wordplay, design inside jokes (baseline shift puns)
- Structure: List-heavy, white space respected

**Trigger Words (Resonate):**
- "Design system," "consistency," "components," "scalable," "flexible," "expressive"
- "Pixel-perfect," "responsive," "accessible," "semantic"
- "Your craft," "elevate," "beautiful," "harmonious"

**Alienation Words (Avoid):**
- "No coding needed" (implies design is simple, undervalues their work)
- "Faster than Figma" (directly competitive, sounds boastful)
- "Just drag and drop" (reductive, misses design nuance)
- "For everyone" (designers want specialized tool, not generic)
- "AI-powered" (unless demonstrably useful; often feels like hype)

**Platforms:**
- Twitter/X (design discourse)
- Dribbble (portfolio sharing)
- Instagram (visual inspiration)
- LinkedIn (less active, but B2B deal-makers there)

**Copy Examples (English):**

1. "Your design system, everywhere it matters"
   - Respects systems thinking; promise of consistency

2. "Components that scale with your craft"
   - "Craft" signals respect; "scale" signals systems

3. "Design faster without compromising detail"
   - Acknowledges tension between speed + quality (designer's real concern)

4. "Responsive by default"
   - Technical, precise language respected by designers

5. "Semantic color, every time"
   - Deep design knowledge signal; appeals to design maturity

**Anti-Patterns (English):**

1. "So easy, even non-designers can use it"
   - Implies design is simple; designer feels dismissed

2. "We built this FOR designers"
   - Sounds patronizing; show, don't tell

3. "Make designs in minutes"
   - Undervalues design craft; sounds AI-generated

4. "Perfect for startups and designers"
   - Bundles designers with non-creative startups; hierarchy mistake

5. "No design skills required"
   - Actively alienates target audience

**Copy Examples (Korean - C8 Mode):**

1. "디자인 시스템, 한 곳에서 관리해"
   - Native Korean + 해체 (management efficiency signal)

2. "컴포넌트가 자라나는 걸 봐"
   - Poetic, growth metaphor; "봐" (look) = invitation

3. "디테일은 그대로, 속도는 빨라져"
   - Addresses speed/quality tension; 는데 structure for flow

4. "동료 디자이너가 있는 거 같아"
   - Implies design community, kinship (반말 + 는데)

5. "새로운 컬러, 매번 헷갈리지 마"
   - Particle-heavy for rhythm; negative reframed as solved

### Persona 2: Developer (Builder, CLI-Native, Precision-First)

**Demographics:**
- Age: 22-50, 85% male (demographic disparity in tech)
- Tools: VSCode, GitHub, terminal, API docs
- Income: $80K-$200K+ (employed to startup founder)
- Location: Distributed, anywhere with good internet

**Psychographics:**
- Values efficiency (keystrokes matter)
- Problem-solver mentality (debug before discuss)
- Documentation-driven (read specs before trying)
- Skeptical of marketing (shows, don't tell)
- Community-oriented (GitHub, Stack Overflow, Discord)

**Copy Preferences:**
- Length: Dense, technical (20-30 words OK if precise)
- Vocabulary: Technical, no hand-waving (API, SDK, latency, throughput)
- Tone: Direct, factual, sometimes dry/funny
- Humor: Dad jokes, tech puns, honest snark
- Structure: Clear hierarchy, code examples trump prose

**Trigger Words (Resonate):**
- "Open source," "documented," "zero-config," "DX" (developer experience)
- "Type-safe," "async," "headless," "stateless," "immutable"
- "Extensible," "composable," "production-ready," "battle-tested"

**Alienation Words (Avoid):**
- "Simple" (without proof; developers want honest difficulty level)
- "Intuitive" (show intuitive design, don't claim it)
- "Game-changing" (feels like marketing hype)
- "No code required" (if they're developers, they code; don't insult their identity)
- "Just use our UI" (developers want programmatic control)

**Platforms:**
- GitHub (code + community)
- Hacker News (news + discussion)
- Reddit (r/webdev, r/golang, domain-specific)
- Twitter/X (tech discourse, rare)

**Copy Examples (English):**

1. "Zero-config setup, full control under the hood"
   - Balances ease (zero-config) + power (control)

2. "TypeScript from the start"
   - Type safety = trust, signals maturity

3. "Open source. MIT licensed. Your rules."
   - Respects developer autonomy; concrete terms

4. "1.3KB gzipped, tree-shakeable"
   - Specific metrics; shows optimization discipline

5. "Deploy anywhere. No vendor lock-in."
   - Addresses real fear (lock-in); respects agency

**Anti-Patterns (English):**

1. "Developers will LOVE this!"
   - Condescending; let developers decide

2. "We simplified it so you don't have to think"
   - Implies developers don't want to think (patronizing)

3. "Powered by AI" (without substance)
   - Vague; developers demand specifics

4. "Join thousands of happy developers"
   - Testimonial-heavy; developers want data/stats

5. "Perfect for beginners and pros"
   - Implies something for everyone = nothing for anyone (developer skepticism)

**Copy Examples (Korean - C8 Mode):**

1. "설정 없이 바로 쓰고, 원할 땐 건드려"
   - Native Korean default; "건드려" = customize freely

2. "타입 안전한 코드가 기본"
   - Straightforward; no marketing fluff

3. "깃허브에 올려져있어. MIT 라이선스"
   - Concrete facts; 해체 casualness appropriate

4. "1.3KB, 트리-셰이크 가능"
   - Technical terms straight; no sugar

5. "어디든 배포돼. 벤더 락인 없어"
   - Parallel structure; 해체 authority tone

### Persona 3: Video Creator (YouTube/TikTok, Energetic, Meme-Literate)

**Demographics:**
- Age: 18-35, 50/50 male/female
- Tools: OBS, Adobe Premiere, DaVinci, CapCut, TikTok Creator Studio
- Income: $20K-$500K (varies: hobby to full-time)
- Location: Everywhere (platform-agnostic)

**Psychographics:**
- Trend-responsive (catches memes within 24hrs)
- Community-building focus (YouTube channel = extension of identity)
- Metric-obsessed (watch time, CTR, retention)
- Authentic voice (audience punishes fakeness)
- Always-on (content ideas 24/7)

**Copy Preferences:**
- Length: Very short, punchable (3-7 words ideal)
- Vocabulary: Current slang, platform-specific terms (algorithm, virality, drop)
- Tone: Energetic, casual, sometimes irreverent
- Humor: Meme language, in-jokes, surprise/relief rhythm
- Structure: Hook first, details never (if hook doesn't land, person leaves)

**Trigger Words (Resonate):**
- "Algorithm," "viral," "CTR," "retention," "thumbnail," "drop," "collab"
- "Authentic," "real," "unfiltered," "trending," "hustle," "grind," "creator economy"
- "Monetize," "grow," "community," "vibe," "energy," "moment"

**Alienation Words (Avoid):**
- "Professional" (feels corporate, anti-creator energy)
- "Optimize" (feels reductive; creators want to express, not calculate)
- "Leverage" (business-speak, kills vibe)
- "Content" (generic; most creators call it "videos" or "streams")
- "Strategy" (implies corporate planning; creators want creative freedom)

**Platforms:**
- TikTok (primary trend source)
- YouTube (community building, monetization)
- Instagram Reels (secondary distribution)
- Discord (creator community, behind-the-scenes)
- Twitter/X (discourse, meme sharing)

**Copy Examples (English):**

1. "Create the video, not the stress"
   - Addresses real pain (editing time); casual tone

2. "Trending sounds? Already in there"
   - Respects trend-responsive nature; "already in there" = fast

3. "Edit faster, actually creative"
   - Balances speed + art (both matter to video creators)

4. "Your vibe, amplified"
   - Respects authenticity; "amplified" = reach without changing voice

5. "Watch your CTR go 📈"
   - Emoji + metric focus; casual celebration

**Anti-Patterns (English):**

1. "Optimize your content pipeline"
   - Too corporate; kills creative energy

2. "Professional-grade editing"
   - "Professional" feels like gatekeeping; creators want peer-level tools

3. "Increase your metrics"
   - Too utilitarian; creators feel reduced to numbers

4. "For serious YouTubers"
   - Gatekeeping; alienates aspiring creators

5. "Make content faster than ever"
   - Vague; no sense of what "faster" means in creator context

**Copy Examples (Korean - C8 Mode):**

1. "영상, 빨리 끝내고 올려"
   - Native Korean urgency; 해체 casual

2. "트렌딩 사운드 자동으로 들어있어"
   - Speaks to real workflow; 해체 convenience signal

3. "편집은 쉽게, 창의력은 더"
   - Parallel structure; acknowledges both needs

4. "너의 톤, 그대로 크게"
   - Respects authenticity; short, punchy

5. "유튜브 알고리즘이 자동으로 잡아"
   - Platform-specific; 해체 confidence

### Persona 4: Writer/Artist (Literary, Poetic, Whitespace-Loving)

**Demographics:**
- Age: 28-55, 60% female
- Tools: Notion, iA Writer, Obsidian, Adobe InDesign
- Income: $30K-$150K (freelance to publishing)
- Location: Urban + college towns, some remote

**Psychographics:**
- Values depth over speed (long-form thinkers)
- Aesthete (cares about typography, formatting, beauty)
- Anti-hype (skeptical of trending, trusts craft)
- Introspective (personal brand = body of work)
- Community-building (literary circles, writing groups)

**Copy Preferences:**
- Length: Long-form acceptable if thoughtful (can be 30+ words if each word counts)
- Vocabulary: Literary, specific, sometimes archaic
- Tone: Contemplative, honest, sometimes wry
- Humor: Witty, understated (not laugh-out-loud)
- Structure: White space respected; paragraphs matter; rhythm in prose

**Trigger Words (Resonate):**
- "Craft," "clarity," "voice," "presence," "depth," "substance"
- "Thought," "reflection," "intention," "consider," "resonate," "authentic"
- "Typography," "composition," "breathing room," "edit ruthlessly," "perfectionist"

**Alienation Words (Avoid):**
- "Easy" (suggests writing is simple; it's not)
- "Churn out" (implies quantity over quality; anti-writer)
- "Viral" (mercenary framing; writers distrust virality)
- "Influencer" (contradicts writer identity)
- "Just" (diminishes craft; "just write" is tone-deaf)

**Platforms:**
- Substack (long-form, community)
- Medium (literary essays)
- Twitter/X (thoughtful threads, rarely trending)
- Email (direct-to-reader, no algorithm)
- Literary magazines (validation source)

**Copy Examples (English):**

1. "Write what you mean. Clearly."
   - Respects craft; clarity as highest value

2. "Edit. Read aloud. Edit again."
   - Ritual description; shows understanding of writer process

3. "Your voice, unfiltered by algorithm"
   - Respects writer autonomy; algorithm = enemy

4. "Thought made visible"
   - Poetic; elevates writing from task to art

5. "Less noise. Your words matter."
   - Anti-hype positioning; focuses on substance

**Anti-Patterns (English):**

1. "Write faster with AI"
   - Feels like threat to writer craft; displacement fear

2. "Everyone's a writer now"
   - Democratization sounds like devaluation to professional writers

3. "Effortless writing"
   - Insults craft; writers know it's never effortless

4. "Trending topics built in"
   - Commercial angle; writers distrust algorithmic trending

5. "Join our writing community"
   - Vague; writers want literary community, not generic group

**Copy Examples (Korean - C8 Mode):**

1. "쓰고 싶은 말을 정확하게"
   - Precision framing; 해요체 warmth

2. "읽어봐. 다시 고쳐봐. 읽어봐"
   - Ritual repetition (rhythmic); validates editing process

3. "너의 목소리, 있는 그대로"
   - Respects authenticity; 해체 personal

4. "생각이 보여"
   - Poetic; 해체 lyrical

5. "노이즈 없이. 너 글이 중요해"
   - Direct; 반말 intimacy affirms writer value

## Persona Detection Signals: How to Identify Which Persona You're Writing For

**Signal Framework:**

### When Writing for a Known Product/Tool

| Signal | What to Look For | Persona Indicator |
|--------|------------------|------------------|
| **Primary tool named** | "Figma users," "developers," "video creators," "writers" | Direct persona ID |
| **Job title** | "Product designer," "engineer," "YouTuber," "author" | Direct persona ID |
| **Pain point described** | "Slow design iteration," "tedious setup," "editing takes hours," "writer's block" | Matches persona frustration |
| **Workflow mentioned** | "Design system management," "API integration," "recording/editing," "editing/rewriting" | Matches persona workflow |
| **Community reference** | "Design Twitter," "Hacker News," "YouTube algo," "Substack writers" | Matches persona hangout |

### When Audience is Unknown or Mixed

**Ask:**
1. What tool does the primary audience use daily?
2. What's their main frustration with current tools?
3. Where do they hang out (Twitter, Discord, GitHub, TikTok)?
4. What do they optimize for (aesthetics, speed, authenticity, thought)?

**Score the Personas:**

```
If answer mentions design/visual tools:           DESIGNER (+2)
If answer mentions coding/dev tools:            DEVELOPER (+2)
If answer mentions video/streaming:             VIDEO CREATOR (+2)
If answer mentions writing/editing:             WRITER/ARTIST (+2)

If pain point is "speed/efficiency":            DEVELOPER bias (+1)
If pain point is "aesthetics/quality":          DESIGNER bias (+1)
If pain point is "reach/virality":              VIDEO CREATOR bias (+1)
If pain point is "authenticity/voice":          WRITER bias (+1)

If main platform is GitHub/Stack Overflow:      DEVELOPER bias (+1)
If main platform is Dribbble/Design Twitter:   DESIGNER bias (+1)
If main platform is TikTok/YouTube:            VIDEO CREATOR bias (+1)
If main platform is Substack/Medium:           WRITER bias (+1)
```

**Decision:** Highest score persona gets 60% tone weight, others get 40% blend.

## Persona Mixing Strategy: Multi-Persona Audience

### When Audience Is Mixed (e.g., Design Tool Used by Designers + Developers)

**Optimization Rule:** Don't try to hit everyone equally. Optimize for primary persona (60%), blend secondary (40%).

### Approach 1: Separate Copy Paths (Recommended)

- Create variant A1: Designer-focused headline
- Create variant A1: Developer-focused headline
- A/B test both; data tells you primary audience
- Ship winner, keep loser for reference

**Example (Design + Dev Tool):**

**Variant: Designer-Focused**
```
"Design systems made beautiful"
- "beautiful" signals aesthetic priority
- "design systems" signals designer sophistication
```

**Variant: Developer-Focused**
```
"Design systems made programmable"
- "programmable" signals API-first, extensibility
- "design systems" speaks to shared concern
```

**Decision:** A/B test. If designers are primary (higher CTR on Variant 1), ship that. If developers primary, ship Variant 2.

### Approach 2: Blended Copy (For When Separate Variants Not Feasible)

**Priority:** Designer (60%) + Developer (40%)

**Example:**
```
"Design systems that scale with your team"
- "Design systems" = designer language
- "scale" = developer concern (growth, extensibility)
- Tone: Professional-but-not-stuffy (respects both)
- No persona-specific inside jokes
```

**Blending Rules:**
- Lead with primary persona language
- Nest secondary persona benefit in supporting sentence
- Avoid alienation words for both
- Tone should be neutral-professional (lose some personality, gain some audience)

### Approach 3: Persona Segmentation (Advanced)

If you have audience data:
- Send Designer-optimized copy to designer segment
- Send Developer-optimized copy to developer segment
- Requires tracking customer segmentation (by job title or tool usage)

## Universal Mode: General Persona Framework (Non-Creator Products)

**When not writing for C8/creator audience, use this framework:**

### 4 Universal Personas

#### 1. B2B Decision Maker (Executive/Buyer)

**Copy focus:** ROI, risk reduction, team impact
- Tone: Confident, authoritative, data-driven
- Length: Short (decision-makers scan)
- Trigger words: "ROI," "risk," "team," "scale," "proven"
- Alienation: "complicated," "requires training"

**Copy example:**
```
"Enterprise teams cut deployment time by 60%"
- Metric-driven
- "enterprise" = tier signal
- "cut" = positive action verb
```

#### 2. B2C Consumer (Everyday User)

**Copy focus:** Problem solved, ease of use, emotional benefit
- Tone: Friendly, casual, reassuring
- Length: Medium (scannable but personal)
- Trigger words: "simple," "time," "life," "easier," "finally"
- Alienation: "technical," "jargon," "complex"

**Copy example:**
```
"Finally, bills that don't confuse you"
- "Finally" = validation of frustration
- "don't confuse you" = emotional, specific benefit
```

#### 3. Technical User (Implementation Focus)

**Copy focus:** Specs, integration, customization
- Tone: Precise, direct, technical
- Length: Longer OK if dense
- Trigger words: "API," "integrate," "customize," "documentation"
- Alienation: "simple," "beginner," "just"

**Copy example:**
```
"REST API, webhooks, custom fields, real-time sync"
- Technical specification
- Each element addresses implementation concern
```

#### 4. Non-Technical User (Ease Focus)

**Copy focus:** No skills needed, visual interface, support available
- Tone: Warm, encouraging, patient
- Length: Medium, with step clarity
- Trigger words: "no skills," "easy," "help," "step-by-step"
- Alienation: "code," "technical," "advanced"

**Copy example:**
```
"Build your site with clicks, not code"
- "no skills needed" implicit in "clicks"
- Respects non-technical comfort level
```

## Final Checkpoint

Before writing copy for a persona-driven product:

1. **Persona ID:** Which of 4 personas (Designer/Dev/Video/Writer) or universal (B2B/B2C/Tech/Non-Tech)?
2. **Primary Signal:** What's the biggest clue (tool, platform, pain point)?
3. **Tone Alignment:** Does my tone match persona preferences?
4. **Language Check:** Am I using trigger words? Avoiding alienation words?
5. **Rhythm:** Does it sound natural for this persona?
6. **Anti-Pattern Scan:** Any red flags that would make this persona uncomfortable?
7. **Mixed Audience?:** Should I create separate variants or blend?

If any fails → adjust, test, then ship.
