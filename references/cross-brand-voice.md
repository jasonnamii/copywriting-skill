# Cross-Brand Voice: Unifying Copy Across All 27 Types

## Part 1: C8 Brand Voice Definition

### The "실력 있는 동료 크리에이터" Persona

C8 mode assumes your brand is voice is this specific archetype. Adapt as needed.

**Who is this person?**
- 5-10 years in field (experienced, not gatekeeping)
- Created their own tools, workflows, or IP
- Teaches others via content, mentorship, or code
- No formal MBA/corporate titles (self-made)
- Fiercely independent but collaborative with peers
- Optimizes for craft, impact, and speed (not prestige)

**How they speak:**
- Direct and opinionated (strong POV)
- Uses insider language without being exclusionary
- Laughs at industry BS, including their own mistakes
- Shares unpolished work-in-progress thinking
- Mentors generously; expects others to ship, not perfect

**Their relationship model:**
- Speaks TO peers, not DOWN to learners
- Assumes audience is competent until proven otherwise
- Values reciprocity: I help you, you help others
- Skeptical of hype; trusts shipped work

**Knowledge level they project:**
- "I know this works because I did it"
- "I tried X, Y, Z; here's what worked"
- "I'm still figuring this out, join me"
- Avoids abstract theory; roots everything in practice

## C8 Voice DO/DON'T Matrix with Examples

### DO ✓

| DO | Example |
|-----|---------|
| **Use first-person plural ("we built this")** | "We spent 6 months cutting unnecessary UI. Here's what we learned." |
| **Name the problem frankly** | "Email marketing tools are bloatware. This one isn't." |
| **Share the constraint/tradeoff** | "We chose speed over perfect. You get results in 30 min, not 2 hours." |
| **Use specific numbers, not superlatives** | "3x faster", not "blazingly fast" |
| **Admit mistakes publicly** | "We shipped this wrong first. Here's v2." |
| **Use casual contractions** | "we've", "it's", "you'll" |
| **Reference creator culture unprompted** | "Shipped > perfect. Iteration > planning." |
| **Assume competence** | "If you know CSS, you can customize in 5 min." |
| **Celebrate unfinished work** | "Alpha. Unstable. But it works." |

### DON'T ✗

| DON'T | Example |
|-------|---------|
| **Overclaim expertise** | "The #1 solution for creators" (let users say it) |
| **Use corporate jargon** | "Synergize", "leverage", "maximize ROI" |
| **Apologize for being different** | "Not like other tools, sorry!" (own it) |
| **Name competitors** | "Better than Asana" (ship and let work speak) |
| **Use hype words without data** | "Revolutionary", "Game-changing" (show, don't tell) |
| **Hedge with "maybe" or "perhaps"** | "This might help you" (commit or don't) |
| **Speak in third person** | "The company believes..." (say "we") |
| **Pretend perfection** | "Flawless integration" (always has edge cases) |

## Korean vs English Voice Differences (Re-creation, Not Translation)

### Korean C8 Voice Pattern

**Structure:** 주어 생략 + 한자어 + 명확한 판단

Example English: "We spent 3 months removing features nobody used."
Example Korean: "쓸모없는 기능 6개 쳤다. 3개월 걸렸다. 결과물은 훨씬 낫다."
(Not: "우리는 아무도 사용하지 않던 6개의 기능을 제거했습니다.")

**Why it works:**
- Drops 주어 (we) for punchiness
- Uses 한자어 (결과물, 기능, 축소) for authority
- Short declarative sentences (존댓말 X, 반말 X → 높낮이 중립)
- Direct judgment without hedge

### English C8 Voice Pattern

**Structure:** Active voice + specific detail + first person plural

Example: "We removed 6 features. Spent 3 months. Shipped 2x faster."

**Why it works:**
- Short, punchy sentences (Western style)
- Active voice, specific numbers
- "We" (collective credit, not CEO halo)
- No hedging language

### Key Differences Summary

| Dimension | Korean | English |
|-----------|--------|---------|
| **Subject** | 생략 (implied) | Explicit (we/you) |
| **Formality** | 반말 느낌 (but 높낮이 중립) | Casual contractions |
| **Emphasis** | 어순 (verb-first or object-first) | Specific nouns + numbers |
| **Authority** | 한자어 (심화, 최적화) | Jargon avoidance |
| **Pacing** | 짧은 문장 + 점 | Short sentences + periods |

## Voice Consistency Audit Checklist

Run this checklist before shipping any copy across 27 types:

**Vocabulary Consistency**
- [ ] Same domain terminology used consistently (e.g., "creator", "ship", "iterate")
- [ ] No contradictory jargon (don't say "move fast" in one type and "ensure quality" in another)
- [ ] Abbreviations (API, MVP, ROI) used consistently or explained first time

**Personality Consistency**
- [ ] Tone of authority is consistent (don't flip from expert to peer)
- [ ] Opinion level consistent (all types equally opinionated or all neutral)
- [ ] Humor style consistent (puns vs. self-deprecation; same tone across types)

**Relationship Model Consistency**
- [ ] Address audience same way (always "you", never "users")
- [ ] Assumption of knowledge consistent (never dumb down in one type, assume genius in another)
- [ ] Permission level consistent (do we ask before claiming they want something?)

**Emotional Consistency**
- [ ] Primary emotion recognizable across 27 types (warmth, urgency, play)
- [ ] No tone whiplash (don't go from irreverent to formal mid-page)
- [ ] Energy level consistent (all types similarly energetic or measured)

**Cross-Language Consistency (if using Korean + English)**
- [ ] Same meaning, not word-for-word translation
- [ ] Personality traits present in both languages
- [ ] No lost nuance (e.g., Korean sarcasm doesn't become English sarcasm differently)

## Voice Adaptation Rules: Flexing Without Breaking

### Core Voice Immutables (Fixed Across All 27 Types)

These traits are ALWAYS present, regardless of type:
- Personality archetype (e.g., "irreverent challenger")
- Opinion level (low/medium/high)
- Key vocabulary (your unique terms)
- Relationship assumption (peers vs expert)
- Emotional primary tone

### Voice Flexibles (Adjust by Type)

These traits adjust per copy type while maintaining identity:
- **Formality:** A5 Error Message can be casual; C10 PR Headline can be formal
- **Humor:** C6 Social Post gets more humor; A5 Error Message gets strategic humor
- **Detail level:** A2 Onboarding can be step-by-step; C3 Landing Page Hero is big-picture
- **Enthusiasm:** A7 Push can be +20 enthusiastic; D2 Re-engagement is -10 enthusiastic

### The Voice Flex Formula

```
Base Brand Voice + (Type Tone Adjustment) = Adapted Voice

Example:
Base: Irreverent Challenger, opinionated, assumes competence
Type C6 Tone: +20 Funny, +15 Casual
Result: Much more jokey and casual, but still irreverent and opinionated

Type A5 Tone: -20 Enthusiastic, -10 Funny (error message)
Result: Same irreverent POV, but sympathetic and direct, less jokey
```

### Audit: Consistency Across Types

After writing copy in 3 types (e.g., C3 Landing Page + A7 Push + D1 Welcome), ask:
- "Would I recognize this brand in each type?" (Yes = ✓)
- "Do all 3 have the same personality, just dressed differently?" (Yes = ✓)
- "Could I mix these up if they were anonymous?" (No = ✓)

## Multi-Channel Voice Lock

When running a campaign across email + push + social:
- **Message** changes (brevity, context)
- **Tone** changes slightly (±10 per dimension)
- **Voice** stays identical

Example Campaign: Feature launch

**Email Subject (C1):** "Wevideo just shipped [feature]. Here's why you needed it."
**Push (A7):** "Shipped: [feature] 🎬 Try it →"
**Social Post (C6):** "We spent 4 months building this. Here's what changed."

All 3 sound like the same person, different contexts.

## Voice Decay Warning Signs

If you see this in your copy, voice is breaking:

| Warning Sign | Fix |
|---|---|
| Sudden formality spike in one type | Audit word choice; swap jargon for plain English |
| Humor disappears in certain types | Audit tone grid; add humor intentionally (not accidentally) |
| Personality unrecognizable in 1+ types | Back to brand canvas; rewrite that type from scratch |
| Contradictory opinions stated | Choose one POV; cascade to all 27 types |
| Different voice in different languages | Recreate in each language, don't translate |
